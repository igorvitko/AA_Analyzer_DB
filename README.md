# AA-Analyzer-DB

Проєкт `AA-Analyzer-DB` збирає дані повітряних тривог з API `alerts.in.ua`, зберігає 
їх у локальну SQLite базу та формує звіт в Excel за вказаний період.

## Що робить проєкт

- Завантажує свіжі дані тривог за списком регіонів з `settings.py`
- Зберігає повідомлення у SQLite базу `data/alerts_history.db`
- Створює звіт по робочим годинам (з врахування обідньої перерви)
- Зберігає готовий файл Excel у `reports/air_raid_working_hours.xlsx`

## Залежності

Встановіть залежності з `requirements.txt`:

```powershell
py -m pip install -r requirements.txt
```

## Налаштування

1. Створіть файл `.env` в корені проєкту.
2. Запишіть токен API у форматі:

```env
TOKEN=ваш_api_токен
```

3. За потреби змініть параметри у `settings.py`:

- `UIDS_OBL` — список регіонів для завантаження даних
- `TARGET_LOCATIONS` — список `location_uid` для формування звіту
- `WORK_START`, `WORK_END`, `LUNCH_START`, `LUNCH_END` — робочий час та обід
- `TIME_OFFSET_HOURS` — зсув часу (наприклад, +3 для Києва)
- `OUTPUT_FILE` — шлях до формованого Excel звіту

## Як запустити

Запустити увесь процес завантаження даних та формування звіту:

```powershell
py main.py
```

Якщо потрібно лише сформувати звіт з уже завантаженої бази, вимкніть завантаження:

```powershell
py main.py -r false
```

Вказання періоду звіту:

```powershell
py main.py -s 2026-06-01 -e 2026-06-30
```

Якщо параметри `-s` та `-e` не задані, звіт формується за поточний календарний місяць.

## Результат

Після виконання `main.py` буде створено Excel-файл зі сумарними хвилинами тривог у робочий 
час по кожному регіону.

- Початкові дані зберігаються в `data/`
- Звіт зберігається в `reports/`

## Примітки

- Команда `-r false` пропускає фазу завантаження даних з API.
- Якщо БД пуста, звіт не буде сформовано.
- Папки `data` і `reports` створюються автоматично за потреби.
